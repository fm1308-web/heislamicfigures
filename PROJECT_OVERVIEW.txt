# Hooman Library — Islamic Figures App
# Project Overview (last updated: March 2026)

## What this app is
An interactive map and timeline of ~622 significant figures in Islamic history.
Each person appears as a dot on a world map. Clicking a dot opens a biography panel.
The app is a single HTML file with no backend — everything runs in the browser.

## The three files that matter
1. index.html — the entire app (map, UI, filters, biography panel)
2. data/islamic/core.json — all 622 people, lightweight fields only
3. data/islamic/details/*.json — heavy biography data, one file per category

## How data loading works
When the app opens, it loads core.json immediately to draw all the dots.
When the user clicks a person, it loads the relevant details file to show the biography.
This two-stage approach keeps the app fast even as records grow.

## The person record
Each person has two sets of fields:

In core.json (always loaded):
  famous        — unique name, used as primary key everywhere
  type          — e.g. Prophet, Caliph, Jurist, Sufi
  tradition     — e.g. Islamic Jurisprudence, Early Ascetics
  dob / dod     — dates in CE (negative = BCE)
  lat / lng     — map coordinates
  teachers      — array of other famous names (drives the silsila/chain view)
  tags          — array for filter buttons (e.g. "Ashra Mubashshara", "Ahl al-Bayt")
  titles        — honorific titles string
  full          — full Arabic name
  city          — place associated with

In details/{chunk}.json (loaded on click):
  school        — biographical description
  books         — array of works with title/url/note
  parents       — "Father / Mother" string
  children      — string list
  spouses       — string list

## Relationship data
Teacher-student links are stored in the teachers field of core.json.
Each entry is an exact string matching another person's famous field.
The frontend uses these to draw the silsila (chain of transmission) view.
